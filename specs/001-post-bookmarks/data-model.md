# Data Model: Post Bookmarks

**Feature**: 001-post-bookmarks | **Date**: 2026-04-07

## Entities

### Bookmark (new)

Represents a saved relationship between a user and a post.

| Field     | Type     | Description                          |
|-----------|----------|--------------------------------------|
| id        | String   | Unique identifier (UUID)             |
| postId    | String   | Reference to the saved post          |
| userId    | String   | Reference to the user who bookmarked |
| createdAt | DateTime | When the bookmark was created        |

**Constraints**:

- `(userId, postId)` must be unique — a user can only bookmark a given post once.
- `postId` must reference an existing post. If the post is deleted, the bookmark is cascade-deleted.
- `userId` must reference the currently authenticated user.

**Lifecycle**:

- Created: User taps bookmark icon on an unbookmarked post.
- Deleted: User taps bookmark icon on a bookmarked post (toggle off).
- No update state — bookmarks are immutable once created.

### Post (existing — no changes)

| Field     | Type     | Description               |
|-----------|----------|---------------------------|
| id        | String   | Unique identifier         |
| title     | String   | Post title                |
| content   | String   | Post body content         |
| authorId  | String   | Author's user ID          |
| createdAt | DateTime | When the post was created |

### User (existing — no changes)

| Field | Type   | Description        |
|-------|--------|--------------------|
| id    | String | Unique identifier  |
| email | String | User email address |
| name  | String | Display name       |

## Relationships

```
User (1) ──── (N) Bookmark (N) ──── (1) Post
```

- A user has zero or many bookmarks.
- A post has zero or many bookmarks (from different users).
- Each bookmark belongs to exactly one user and exactly one post.

## Supabase Table Schema

### `bookmarks` table (new)

| Column     | Type        | Constraints                                     |
|------------|-------------|-------------------------------------------------|
| id         | uuid        | PRIMARY KEY, DEFAULT gen_random_uuid()          |
| user_id    | uuid        | NOT NULL, FK → auth.users(id) ON DELETE CASCADE |
| post_id    | uuid        | NOT NULL, FK → posts(id) ON DELETE CASCADE      |
| created_at | timestamptz | NOT NULL, DEFAULT now()                         |

**Indexes**:

- PRIMARY KEY on `id`
- UNIQUE on `(user_id, post_id)` — enforces one bookmark per user per post
- INDEX on `user_id` — fast lookups for "get all bookmarks for user"
- INDEX on `post_id` — fast lookups for "is this post bookmarked by user"

**RLS (Row Level Security) Policies**:

- SELECT: Users can only read their own bookmarks (`auth.uid() = user_id`)
- INSERT: Users can only create bookmarks for themselves (`auth.uid() = user_id`)
- DELETE: Users can only delete their own bookmarks (`auth.uid() = user_id`)

## Model ↔ Entity Mapping

### BookmarkModel (data layer)

```
BookmarkModel.fromJson(json) → BookmarkModel
  json['id']         → id (String)
  json['user_id']    → userId (String)
  json['post_id']    → postId (String)
  json['created_at'] → createdAt (DateTime.parse)

BookmarkModel.toJson() → Map<String, dynamic>
  userId  → 'user_id'
  postId  → 'post_id'
  (id and created_at omitted — auto-generated by DB)

BookmarkModel.toEntity() → Bookmark
  Direct field mapping to Bookmark entity
```

## Query Patterns

### Get user's bookmarks (paginated)

```
FROM bookmarks
SELECT *, posts(*)
WHERE user_id = :currentUserId
ORDER BY created_at DESC
RANGE :offset to :offset+19
```

### Add bookmark

```
INSERT INTO bookmarks (user_id, post_id)
VALUES (:currentUserId, :postId)
```

### Remove bookmark

```
DELETE FROM bookmarks
WHERE user_id = :currentUserId AND post_id = :postId
```

### Check if post is bookmarked

```
SELECT id FROM bookmarks
WHERE user_id = :currentUserId AND post_id = :postId
LIMIT 1
```

### Batch check bookmark status for post list

```
SELECT post_id FROM bookmarks
WHERE user_id = :currentUserId AND post_id IN (:postIds)
```
